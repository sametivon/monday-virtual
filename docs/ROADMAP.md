# Roadmap

## MVP — Presence Core (single tenant)
- **M0 Foundation** ✅ — monorepo, shared contracts, Prisma schema, NestJS + Next.js skeletons, Docker/K8s/CI. *(current)*
- **M1 Monday app + auth** ✅ — registered app (Board View + Workspace view), iframe loads web, sessionToken (client-secret-signed) → app JWT, Tenant/User upsert, branding load.
- **M2 3D engine + avatars** ✅ — GLTF avatars (KayKit CC0, self-hosted) + crossfading animation state machine (idle/walk/run/wave/sit; G = wave, X = sit), click-to-move, third-person camera rig, designed Lobby (tables/chairs/screens/whiteboard/portal), object renderer registry.
- **M3 Realtime presence** ✅ — fixed 10Hz server-tick batching (one broadcast + one pipelined Redis flush per space per tick), client-side interpolation (exponential smoothing, shortest-arc yaw, teleport snap), AOI render-culling, dirty-checked sends, presence list panel, join/leave, presence TTL ghost-reaping. *(Server-side cell-based AOI deferred to Phase 3 scale-hardening.)*
- **M4 Proximity media** ✅ — auto-join space voice (listen-only until mic on), Web-Audio spatial graph (distance falloff per scene config + stereo pan by bearing, driven by tick targets so it works in backgrounded tabs), meeting-table full-volume sub-rooms with camera + screen-share tiles, mic carries across rooms, serialized room connects (StrictMode/duplicate-identity safe).
- **M5 Chat** ✅ — Space/Global/DM scopes (tenant room for global, user rooms for DMs), @mention autocomplete from the live roster, mention/DM pings + unread badges, emoji quick-row, fire-and-forget persistence + paginated history endpoint, DMs opened from the presence list, typing guard (chat input never moves the avatar).
- **M6 Polish + perf** ✅ — AOI culling + nameplate LOD + adaptive DPR (PerformanceMonitor), 60fps validated (64fps measured on headless *software* GL; real GPUs higher), [`docs/DEPLOYMENT.md`](DEPLOYMENT.md) (Vercel + Railway/Render + Neon + Upstash). *Deferred: instancing (object counts don't warrant it yet), Sentry/OTel wiring (env-gated, enable when DSNs exist), actual PaaS deploy (needs hosting accounts).*

Backed by `/rbac` (members/roles/assign/role-edit); last-admin + self-demote + admin-can't-drop-role/user-manage guards; `/me` now resolves permissions from the user's current role so changes apply on next load (was reading the stale JWT set). **Per-tenant scene theming shipped**: the tenant's branding accent recolors the scene accent (trim glow, portal, stage lip, hex carpet lines) at manifest build time, with a per-scene `interior.themeFromBranding` opt-out (default on). **Phase 2 COMPLETE.**

## Phase 3 — Events, Analytics, AI, Scale-hardening
- Event mode (conference/workshop/town-hall/training) — **shipped**: admins author events (type/title/time + agenda items + speakers) bound to an auditorium space (`/events` CRUD, EVENT_CREATE/EVENT_MANAGE); members see an events list + RSVP (`/events/:id/register`); presenters flip an event LIVE/ENDED (`/events/:id/go-live`, PRESENT); **attendance auto-marks** when a registered (or walk-in) user joins the event's space while it's LIVE — wired into the realtime gateway's join handler (fire-and-forget, like analytics). Launcher 📅 Events panel: list, RSVP, admin create/edit, presenter Go-Live/End. **Attendance export shipped**: `GET /events/:id/attendance` (JSON) + `GET /events/:id/attendance.csv` (downloadable, EVENT_MANAGE) — every registrant with RSVP time + attended flag; the Events panel shows a ⬇ CSV button per event with registrations. *Event item complete.*
- Analytics dashboard — **shipped**: the realtime gateway captures activity (space_join/leave with session duration, chat_send, reaction, hand_raise) into AnalyticsEvent fire-and-forget; the API aggregates a windowed summary (`GET /analytics/summary?days=`, ANALYTICS_VIEW) — active users, sessions, avg session length, messages/reactions/hands, a daily-active-users series, and per-space engagement; admin dashboard UI (📈 Analytics on the launcher) with 7/30/90-day ranges, KPI chips, a trend chart, and a by-space table. **Occupancy heatmap shipped**: the realtime tick loop samples player positions on a slow timer (OCCUPANCY_SAMPLE_INTERVAL_MS, overridable via OCCUPANCY_SAMPLE_MS) into compact `space_occupancy` AnalyticsEvents; `GET /analytics/heatmap` bins those samples into a normalized per-space grid over the floor bounds; the dashboard's Heatmap tab renders a cool→hot top-down floor plan with a per-space picker. *Remaining: materialized rollups (scale-hardening).*
- AI assistant (OpenAI-compatible abstraction): answer questions, navigate, explain Monday boards, summarize meetings. **SKIPPED — product decision (no AI assistant).**
- Scale hardening — **in progress**: **server-side cell-based AOI shipped + made cross-node-correct** (the deferred M3 item). Crowded spaces (≥ SERVER_AOI_MIN_PLAYERS local players) use **spatial cell rooms**: every socket subscribes to its current AOI cell room (the gateway swaps rooms as players cross cell boundaries), and each mover's tick update is emitted to the 3×3 block of cell rooms around it — so a recipient gets only nearby movers, and because cell rooms are real Socket.IO rooms the Redis adapter fans them correctly ACROSS NODES (a recipient on node B near a mover on node A still gets it, no manual position gathering). Small spaces keep the single room-wide broadcast. Pure cell/interest logic in `aoi.ts`, unit-tested (8 tests). **Load-test harness shipped** (`scripts/load-test.cjs`): N synthetic socket.io clients into one space, drives movement, measures inbound tick rate + payload culling. Verified locally: 100 concurrent clients, all connected, steady 9.6 ticks/s, **mean 17.9 updates/tick/client vs 99 without AOI = ~82% bandwidth cut** (culling tightens as population grows). **GDPR tooling shipped**: `GET /gdpr/users/:id/export(.json)` (full personal-data export, access/portability) + `POST /gdpr/users/:id/erase` (right to be forgotten — scrubs PII, soft-deletes + tombstones mondayUserId so re-login can't resurrect, deletes sessions/DMs, anonymizes room/global messages, detaches analytics, writes an AuditLog, all in one transaction); admin UI on the Team & roles Members tab (per-member Export + type-to-confirm Erase). *Remaining (need infra/accounts): read replicas, analytics partitioning/materialized rollups, self-hosted K8s + LiveKit option, a true distributed 1000-concurrent load run.*
